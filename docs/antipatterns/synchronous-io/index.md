## <a name="how-to-fix-the-problem"></a><span data-ttu-id="3f265-124">A probléma megoldása</span><span class="sxs-lookup"><span data-stu-id="3f265-124">How to fix the problem</span></span>

## <a name="considerations"></a><span data-ttu-id="3f265-143">Megfontolandó szempontok</span><span class="sxs-lookup"><span data-stu-id="3f265-143">Considerations</span></span>

- <span data-ttu-id="3f265-144">A várhatóan nagyon rövid élettartamú és nem versengő I/O műveletek jobb teljesítményt biztosíthatnak szinkron műveletekként.</span><span class="sxs-lookup"><span data-stu-id="3f265-144">I/O operations that are expected to be very short lived and are unlikely to cause contention might be more performant as synchronous operations.</span></span> <span data-ttu-id="3f265-145">Ilyen lehet például egy SSD meghajtón található kisméretű fájlok olvasása.</span><span class="sxs-lookup"><span data-stu-id="3f265-145">An example might be reading small files on an SSD drive.</span></span> <span data-ttu-id="3f265-146">A feladatok másik szálra küldéséhez, majd a feladat befejeztével a szál szinkronizálásához szükséges többletráfordítás esetleg meghaladja az aszinkron I/O működés előnyeit.</span><span class="sxs-lookup"><span data-stu-id="3f265-146">The overhead of dispatching a task to another thread, and synchronizing with that thread when the task completes, might outweigh the benefits of asynchronous I/O.</span></span> <span data-ttu-id="3f265-147">Az ilyen esetek azonban meglehetősen ritkák, és a legtöbb I/O műveletet érdemes aszinkron módon végrehajtani.</span><span class="sxs-lookup"><span data-stu-id="3f265-147">However, these cases are relatively rare, and most I/O operations should be done asynchronously.</span></span>
