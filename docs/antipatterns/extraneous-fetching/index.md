# <a name="extraneous-fetching-antipattern"></a><span data-ttu-id="a4012-103">Felesleges beolvasások kizárási minta</span><span class="sxs-lookup"><span data-stu-id="a4012-103">Extraneous Fetching antipattern</span></span>

## <a name="problem-description"></a><span data-ttu-id="a4012-105">A probléma leírása</span><span class="sxs-lookup"><span data-stu-id="a4012-105">Problem description</span></span>

<span data-ttu-id="a4012-106">Ez a kizárási minta akkor jelentkezik, ha az alkalmazás az I/O-kérések mennyiségének csökkentése érdekében lekéri az összes adatot, amelyre *esetleg* szüksége lehet.</span><span class="sxs-lookup"><span data-stu-id="a4012-106">This antipattern can occur if the application tries to minimize I/O requests by retrieving all of the data that it *might* need.</span></span> <span data-ttu-id="a4012-107">Ez gyakran a [Forgalmas I/O][chatty-io] kizárási minta ellensúlyozásának eredményeképpen jelentkezik.</span><span class="sxs-lookup"><span data-stu-id="a4012-107">This is often a result of overcompensating for the [Chatty I/O][chatty-io] antipattern.</span></span> <span data-ttu-id="a4012-108">Előfordulhat például. hogy az alkalmazás lekéri az adatbázisban található összes termék adatait.</span><span class="sxs-lookup"><span data-stu-id="a4012-108">For example, an application might fetch the details for every product in a database.</span></span> <span data-ttu-id="a4012-109">A felhasználónak azonban csak az adatok egy részhalmazára van szüksége (a többi nem is feltétlenül érinti a felhasználót), és valószínűleg nem is szeretné az *összes* terméket egyszerre megtekinteni.</span><span class="sxs-lookup"><span data-stu-id="a4012-109">But the user may need just a subset of the details (some may not be relevant to customers), and probably doesn't need to see *all* of the products at once.</span></span> <span data-ttu-id="a4012-110">Még ha a felhasználó a teljes katalógusban böngész is, észszerű az eredményeket külön lapokra osztani, például egyszerre csak 20 találat megjelenítésével.</span><span class="sxs-lookup"><span data-stu-id="a4012-110">Even if the user is browsing the entire catalog, it would make sense to paginate the results &mdash; showing 20 at a time, for example.</span></span>

<span data-ttu-id="a4012-111">A probléma másik oka a nem megfelelő programozási és kialakítási gyakorlatokban keresendő.</span><span class="sxs-lookup"><span data-stu-id="a4012-111">Another source of this problem is following poor programming or design practices.</span></span> <span data-ttu-id="a4012-112">A következő kód például az Entity Framework használatával lekéri az összes termék minden adatát.</span><span class="sxs-lookup"><span data-stu-id="a4012-112">For example, the following code uses Entity Framework to fetch the complete details for every product.</span></span> <span data-ttu-id="a4012-113">Ezután szűri az adatokat, és csak a mezők egy részét adja vissza, a többit pedig elveti.</span><span class="sxs-lookup"><span data-stu-id="a4012-113">Then it filters the results to return only a subset of the fields, discarding the rest.</span></span> <span data-ttu-id="a4012-114">A teljes kódmintát [itt][sample-app] találja.</span><span class="sxs-lookup"><span data-stu-id="a4012-114">You can find the complete sample [here][sample-app].</span></span>

```csharp
public async Task<IHttpActionResult> GetAllFieldsAsync()
{
    using (var context = new AdventureWorksContext())
    {
        // Execute the query. This happens at the database.
        var products = await context.Products.ToListAsync();

        // Project fields from the query results. This happens in application memory.
        var result = products.Select(p => new ProductInfo { Id = p.ProductId, Name = p.Name });
        return Ok(result);
    }
}
```

<span data-ttu-id="a4012-115">A következő példában az alkalmazás egy olyan összesítéshez kér le adatokat, amelyet az adatbázis is elvégezhetett volna.</span><span class="sxs-lookup"><span data-stu-id="a4012-115">In the next example, the application retrieves data to perform an aggregation that could be done by the database instead.</span></span> <span data-ttu-id="a4012-116">Az alkalmazás az értékesítések végösszegének kiszámításához lekéri az összes értékesített megrendeléshez tartozó minden bejegyzést, majd ezekből számítja ki a végösszeget.</span><span class="sxs-lookup"><span data-stu-id="a4012-116">The application calculates total sales by getting every record for all orders sold, and then computing the sum over those records.</span></span> <span data-ttu-id="a4012-117">A teljes kódmintát [itt][sample-app] találja.</span><span class="sxs-lookup"><span data-stu-id="a4012-117">You can find the complete sample [here][sample-app].</span></span>

```csharp
public async Task<IHttpActionResult> AggregateOnClientAsync()
{
    using (var context = new AdventureWorksContext())
    {
        // Fetch all order totals from the database.
        var orderAmounts = await context.SalesOrderHeaders.Select(soh => soh.TotalDue).ToListAsync();

        // Sum the order totals in memory.
        var total = orderAmounts.Sum();
        return Ok(total);
    }
}
```

<span data-ttu-id="a4012-118">A következő példa egy kisebb hibát mutat be, amely abból származik, ahogy az Entity Framework a LINQ to Entities műveletet alkalmazza.</span><span class="sxs-lookup"><span data-stu-id="a4012-118">The next example shows a subtle problem caused by the way Entity Framework uses LINQ to Entities.</span></span>

```csharp
var query = from p in context.Products.AsEnumerable()
            where p.SellStartDate < DateTime.Now.AddDays(-7) // AddDays cannot be mapped by LINQ to Entities
            select ...;

List<Product> products = query.ToList();
```

<span data-ttu-id="a4012-119">Az alkalmazás az egy hétnél régebbi `SellStartDate` dátummal rendelkező termékeket keresi.</span><span class="sxs-lookup"><span data-stu-id="a4012-119">The application is trying to find products with a `SellStartDate` more than a week old.</span></span> <span data-ttu-id="a4012-120">A legtöbb esetben a LINQ to Entities a `where` záradékokat az adatbázis által végrehajtandó SQL-utasításokká fordítaná le.</span><span class="sxs-lookup"><span data-stu-id="a4012-120">In most cases, LINQ to Entities would translate a `where` clause to a SQL statement that is executed by the database.</span></span> <span data-ttu-id="a4012-121">Ebben az esetben azonban a LINQ to Entities nem tudja az `AddDays` metódust SQL-re leképezni.</span><span class="sxs-lookup"><span data-stu-id="a4012-121">In this case, however, LINQ to Entities cannot map the `AddDays` method to SQL.</span></span> <span data-ttu-id="a4012-122">Ehelyett a `Product` tábla minden egyes sorát visszaadja, és az eredmények szűrését a memóriában végzi el.</span><span class="sxs-lookup"><span data-stu-id="a4012-122">Instead, every row from the `Product` table is returned, and the results are filtered in memory.</span></span>

<span data-ttu-id="a4012-123">Az `AsEnumerable` meghívása utal arra, hogy itt probléma van.</span><span class="sxs-lookup"><span data-stu-id="a4012-123">The call to `AsEnumerable` is a hint that there is a problem.</span></span> <span data-ttu-id="a4012-124">Ez a metódus az eredményeket egy `IEnumerable` felületté konvertálja.</span><span class="sxs-lookup"><span data-stu-id="a4012-124">This method converts the results to an `IEnumerable` interface.</span></span> <span data-ttu-id="a4012-125">Bár az `IEnumerable` támogatja a szűrést, arra az *ügyféloldalon*, és nem az adatbázisban kerül sor.</span><span class="sxs-lookup"><span data-stu-id="a4012-125">Although `IEnumerable` supports filtering, the filtering is done on the *client* side, not the database.</span></span> <span data-ttu-id="a4012-126">A LINQ to Entities alapértelmezés szerint az `IQueryable` függvényt használja, amely a szűrési feladat felelősségét átadja az adatforrásnak.</span><span class="sxs-lookup"><span data-stu-id="a4012-126">By default, LINQ to Entities uses `IQueryable`, which passes the responsibility for filtering to the data source.</span></span>

## <a name="how-to-fix-the-problem"></a><span data-ttu-id="a4012-127">A probléma megoldása</span><span class="sxs-lookup"><span data-stu-id="a4012-127">How to fix the problem</span></span>

<span data-ttu-id="a4012-128">Kerülje az olyan nagyobb adatmennyiségek lekérését, amelyek elavulhatnak vagy amelyek el lesznek vetve, és csak az épp végrehajtott művelethez szükséges adatokat hívja le.</span><span class="sxs-lookup"><span data-stu-id="a4012-128">Avoid fetching large volumes of data that may quickly become outdated or might be discarded, and only fetch the data needed for the operation being performed.</span></span>

<span data-ttu-id="a4012-129">Ahelyett, hogy valamely tábla összes oszlopát lekérné, majd szűrést futtatna rajtuk, válassza ki a kívánt oszlopokat az adatbázisban.</span><span class="sxs-lookup"><span data-stu-id="a4012-129">Instead of getting every column from a table and then filtering them, select the columns that you need from the database.</span></span>

```csharp
public async Task<IHttpActionResult> GetRequiredFieldsAsync()
{
    using (var context = new AdventureWorksContext())
    {
        // Project fields as part of the query itself
        var result = await context.Products
            .Select(p => new ProductInfo {Id = p.ProductId, Name = p.Name})
            .ToListAsync();
        return Ok(result);
    }
}
```

<span data-ttu-id="a4012-130">Ugyanígy az összesítéseket is az adatbázisban, és ne az alkalmazás memóriájában hajtsa végre.</span><span class="sxs-lookup"><span data-stu-id="a4012-130">Similarly, perform aggregation in the database and not in application memory.</span></span>

```csharp
public async Task<IHttpActionResult> AggregateOnDatabaseAsync()
{
    using (var context = new AdventureWorksContext())
    {
        // Sum the order totals as part of the database query.
        var total = await context.SalesOrderHeaders.SumAsync(soh => soh.TotalDue);
        return Ok(total);
    }
}
```

<span data-ttu-id="a4012-131">Az Entity Framework használatakor bizonyosodjon meg róla, hogy a LINQ-lekérdezéseket az `IQueryable` felület és nem az `IEnumerable` hajtja végre.</span><span class="sxs-lookup"><span data-stu-id="a4012-131">When using Entity Framework, ensure that LINQ queries are resolved using the `IQueryable`interface and not `IEnumerable`.</span></span> <span data-ttu-id="a4012-132">Előfordulhat, hogy módosítania kell a lekérdezést, hogy csak olyan függvényeket használjon, amelyek az adatforrásra vonatkoztathatók.</span><span class="sxs-lookup"><span data-stu-id="a4012-132">You may need to adjust the query to use only functions that can be mapped to the data source.</span></span> <span data-ttu-id="a4012-133">A korábbi példa átdolgozható: ha az `AddDays` metódust eltávolítja a lekérdezésből, a szűrést az adatbázisban lehet végrehajtani.</span><span class="sxs-lookup"><span data-stu-id="a4012-133">The earlier example can be refactored to remove the `AddDays` method from the query, allowing filtering to be done by the database.</span></span>

```csharp

var query = from p in context.Products
            where p.SellStartDate < dateSince // This criterion can be passed to the database by LINQ to Entities
            select ...;

List<Product> products = query.ToList();
```
