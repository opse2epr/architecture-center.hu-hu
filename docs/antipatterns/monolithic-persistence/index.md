Ennek két fő oka van:</span><span class="sxs-lookup"><span data-stu-id="8984e-109">Following the traditional approach and putting all of this information into the same data store can hurt performance, for two main reasons:</span></span>

- <span data-ttu-id="8984e-110">A nagy mennyiségű, egymáshoz nem kapcsolódó adatok egyetlen adattárolóban történő tárolása és lekérése versengést idéz elő, amely lassú válaszidőkhöz és csatlakozási hibákhoz vezet.</span><span class="sxs-lookup"><span data-stu-id="8984e-110">Storing and retrieving large amounts of unrelated data in the same data store can cause contention, which in turn leads to slow response times and connection failures.</span></span>

## <a name="how-to-fix-the-problem"></a><span data-ttu-id="8984e-117">A probléma megoldása</span><span class="sxs-lookup"><span data-stu-id="8984e-117">How to fix the problem</span></span>

<span data-ttu-id="8984e-118">Válassza szét az adatokat a felhasználásuk módja szerint.</span><span class="sxs-lookup"><span data-stu-id="8984e-118">Separate data according to its use.</span></span> <span data-ttu-id="8984e-119">Minden egyes adatkészlethez külön adattárolót válasszon ki, amely a legalkalmasabb az adott adat felhasználási módjához.</span><span class="sxs-lookup"><span data-stu-id="8984e-119">For each data set, select a data store that best matches how that data set will be used.</span></span> <span data-ttu-id="8984e-120">Az előző példában az alkalmazás naplóját el kell különíteni az üzleti adatokat tartalmazó adatbázistól:</span><span class="sxs-lookup"><span data-stu-id="8984e-120">In the previous example, the application should be logging to a separate store from the database that holds business data:</span></span>

```csharp
    private static readonly string ProductionDb = ...;
    private static readonly string LogDb = ...;

    public async Task<IHttpActionResult> PostAsync([FromBody]string value)
    {

    }

```

## <a name="considerations"></a><span data-ttu-id="8984e-121">Megfontolandó szempontok</span><span class="sxs-lookup"><span data-stu-id="8984e-121">Considerations</span></span>
