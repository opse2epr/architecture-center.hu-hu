<span data-ttu-id="77ba6-115">Roy Fielding 2000-ben mutatta be a REST (Representational State Transfer, reprezentáción alapuló állapotátvitel) nevű, a webes szolgáltatások tervezésére szolgáló architekturális módszert.</span><span class="sxs-lookup"><span data-stu-id="77ba6-115">In 2000, Roy Fielding proposed Representational State Transfer (REST) as an architectural approach to designing web services.</span></span> <span data-ttu-id="77ba6-116">A REST egy architekturális stílus a hipermédián alapuló elosztott rendszerek készítéséhez.</span><span class="sxs-lookup"><span data-stu-id="77ba6-116">REST is an architectural style for building distributed systems based on hypermedia.</span></span> <span data-ttu-id="77ba6-117">A REST mindennemű mögöttes protokolltól független, és nem feltétlenül kötődik a HTTP-hez.</span><span class="sxs-lookup"><span data-stu-id="77ba6-117">REST is independent of any underlying protocol and is not necessarily tied to HTTP.</span></span> <span data-ttu-id="77ba6-118">A leggyakoribb REST-alkalmazások azonban a HTTP-protokollt használják, így ez az útmutató elsősorban a HTTP REST API-k tervezésére koncentrál.</span><span class="sxs-lookup"><span data-stu-id="77ba6-118">However, most common REST implementations use HTTP as the application protocol, and this guide focuses on designing REST APIs for HTTP.</span></span>

<span data-ttu-id="77ba6-433">A verziószám lekérdezésisztring-paramétereként való feltüntetése helyett használhat egy egyéni fejlécet, amely jelzi az erőforrás verzióját.</span><span class="sxs-lookup"><span data-stu-id="77ba6-433">Rather than appending the version number as a query string parameter, you could implement a custom header that indicates the version of the resource.</span></span> <span data-ttu-id="77ba6-434">E módszerhez szükség van arra, hogy az ügyfélalkalmazás a megfelelő fejlécet adja hozzá minden kéréshez, ugyanakkor az ügyfélkérést kezelő programkód használhat alapértelmezett értéket (1-es verzió), ha a verziófejléc ki van hagyva.</span><span class="sxs-lookup"><span data-stu-id="77ba6-434">This approach requires that the client application adds the appropriate header to any requests, although the code handling the client request could use a default value (version 1) if the version header is omitted.</span></span> <span data-ttu-id="77ba6-435">Az alábbi példákban a *Custom-Header* nevű egyéni fejlécet fogjuk használni.</span><span class="sxs-lookup"><span data-stu-id="77ba6-435">The following examples utilize a custom header named *Custom-Header*.</span></span> <span data-ttu-id="77ba6-436">A fejléc értéke a webes API verzióját adja meg.</span><span class="sxs-lookup"><span data-stu-id="77ba6-436">The value of this header indicates the version of web API.</span></span>

### <a name="media-type-versioning"></a><span data-ttu-id="77ba6-440">Adathordozótípus-verziókezelés</span><span class="sxs-lookup"><span data-stu-id="77ba6-440">Media type versioning</span></span>

<span data-ttu-id="77ba6-441">Amikor egy ügyfélalkalmazás HTTP GET-kérést küld egy webkiszolgálónak, akkor közölnie kell az általa kezelt tartalomformátumokat egy Accept fejléc használatával, ahogy ezt az útmutató korábbi szakaszaiban is ismertettük.</span><span class="sxs-lookup"><span data-stu-id="77ba6-441">When a client application sends an HTTP GET request to a web server it should stipulate the format of the content that it can handle by using an Accept header, as described earlier in this guidance.</span></span> <span data-ttu-id="77ba6-442">Az *Accept* fejléc célja sokszor az, hogy lehetővé tegye az ügyfélalkalmazás számára annak meghatározását, hogy a választörzs XML, JSON vagy valamilyen más gyakori formátummal bír, amelynek elemzésére az ügyfél képes.</span><span class="sxs-lookup"><span data-stu-id="77ba6-442">Frequently the purpose of the *Accept* header is to allow the client application to specify whether the body of the response should be XML, JSON, or some other common format that the client can parse.</span></span> <span data-ttu-id="77ba6-443">Lehetséges azonban egyéni adathordozó-típusok meghatározása is.
