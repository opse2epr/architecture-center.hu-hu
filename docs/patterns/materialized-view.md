<span data-ttu-id="2eb09-105">Létrehozhat előre kitöltött nézeteket egy vagy több adattár adataiból, ha az adatok formázása nem ideális a szükséges lekérdezési műveletekhez.</span><span class="sxs-lookup"><span data-stu-id="2eb09-105">Generate prepopulated views over the data in one or more data stores when the data isn't ideally formatted for required query operations.</span></span> <span data-ttu-id="2eb09-106">Ez segíthet a hatékony lekérdezésben és adatkinyerésben, valamint az alkalmazás teljesítményének javításában.</span><span class="sxs-lookup"><span data-stu-id="2eb09-106">This can help support efficient querying and data extraction, and improve application performance.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="2eb09-107">Kontextus és probléma</span><span class="sxs-lookup"><span data-stu-id="2eb09-107">Context and problem</span></span>

<span data-ttu-id="2eb09-108">Az adatok tárolásakor a fejlesztők és adat-rendszergazdák gyakran az adatok tárolásának módjára, nem pedig az írás módjára összpontosítanak.</span><span class="sxs-lookup"><span data-stu-id="2eb09-108">When storing data, the priority for developers and data administrators is often focused on how the data is stored, as opposed to how it's read.</span></span> <span data-ttu-id="2eb09-109">A kiválasztott tárolási formátum általában szorosan összefügg az adatok formátumával, az adatok méret- és integritáskezelési követelményeivel és a használt tároló típusával.</span><span class="sxs-lookup"><span data-stu-id="2eb09-109">The chosen storage format is usually closely related to the format of the data, requirements for managing data size and data integrity, and the kind of store in use.</span></span> <span data-ttu-id="2eb09-110">NoSQL-dokumentumtároló használata esetén például az adatok gyakran szerepelnek összesítések sorozataként, és mindegyik sorozat minden információt tartalmaz az adott entitásról.</span><span class="sxs-lookup"><span data-stu-id="2eb09-110">For example, when using NoSQL document store, the data is often represented as a series of aggregates, each containing all of the information for that entity.</span></span>

<span data-ttu-id="2eb09-111">Azonban ez negatív hatással lehet a lekérdezésekre.</span><span class="sxs-lookup"><span data-stu-id="2eb09-111">However, this can have a negative effect on queries.</span></span> <span data-ttu-id="2eb09-112">Ha egy lekérdezésnek bizonyos entitások adatainak egy részére van szüksége (például néhány ügyfél rendeléseinek egyes részletei), a kapcsolódó entitás összes adatát ki kell nyernie a szükséges információ beszerzéséhez.</span><span class="sxs-lookup"><span data-stu-id="2eb09-112">When a query only needs a subset of the data from some entities, such as a summary of orders for several customers without all of the order details, it must extract all of the data for the relevant entities in order to obtain the required information.</span></span>

## <a name="solution"></a><span data-ttu-id="2eb09-113">Megoldás</span><span class="sxs-lookup"><span data-stu-id="2eb09-113">Solution</span></span>

<span data-ttu-id="2eb09-114">A hatékony lekérdezések támogatásához gyakori megoldás egy olyan nézet létrehozása, amely a szükséges eredménykészlethez megfelelő formátumban materializálja az adatokat.</span><span class="sxs-lookup"><span data-stu-id="2eb09-114">To support efficient querying, a common solution is to generate, in advance, a view that materializes the data in a format suited to the required results set.</span></span> <span data-ttu-id="2eb09-115">A Materialized View minta előre feltöltött adatnézetek létrehozását jelenti az olyan környezetekben, ahol a forrásadatok nem a lekérdezésnek megfelelő formátumban vannak, a megfelelő lekérdezés létrehozása bonyolult, vagy a lekérdezés teljesítménye nem megfelelő az adatok vagy az adattár természete miatt.</span><span class="sxs-lookup"><span data-stu-id="2eb09-115">The Materialized View pattern describes generating prepopulated views of data in environments where the source data isn't in a suitable format for querying, where generating a suitable query is difficult, or where query performance is poor due to the nature of the data or the data store.</span></span>

<span data-ttu-id="2eb09-116">Ezek a materializált nézetek, amelyek csak a lekérdezéshez szükséges adatokat tartalmazzák, lehetővé teszik, hogy az alkalmazások gyorsan beszerezzék a számukra szükséges információkat.</span><span class="sxs-lookup"><span data-stu-id="2eb09-116">These materialized views, which only contain data required by a query, allow applications to quickly obtain the information they need.</span></span> <span data-ttu-id="2eb09-117">A táblázatok összekapcsolása és az adatentitások kombinálása mellett a materializált nézetek tartalmazhatják a számított oszlopok vagy adatelemek aktuális értékeit, az értékek kombinálásának vagy adatelemek átalakításának eredményeit, illetve a lekérdezés részeként megadott értékeket is.</span><span class="sxs-lookup"><span data-stu-id="2eb09-117">In addition to joining tables or combining data entities, materialized views can include the current values of calculated columns or data items, the results of combining values or executing transformations on the data items, and values specified as part of the query.</span></span> <span data-ttu-id="2eb09-118">A materializált nézet egyetlen lekérdezéshez is optimalizálható.</span><span class="sxs-lookup"><span data-stu-id="2eb09-118">A materialized view can even be optimized for just a single query.</span></span>

<span data-ttu-id="2eb09-119">A lényeg az, hogy a materializált nézet és a benne található adatok teljesen pótolhatók, mivel teljesen újraépíthetők a forrásadattárakból.</span><span class="sxs-lookup"><span data-stu-id="2eb09-119">A key point is that a materialized view and the data it contains is completely disposable because it can be entirely rebuilt from the source data stores.</span></span> <span data-ttu-id="2eb09-120">A materializált nézetet soha nem közvetlenül egy alkalmazás frissíti, ezért ez egy specializált gyorsítótár.</span><span class="sxs-lookup"><span data-stu-id="2eb09-120">A materialized view is never updated directly by an application, and so it's a specialized cache.</span></span>

<span data-ttu-id="2eb09-121">Ha a nézet forrásadatai megváltoznak, az új információk megjelenítéséhez frissíteni kell a nézetet.</span><span class="sxs-lookup"><span data-stu-id="2eb09-121">When the source data for the view changes, the view must be updated to include the new information.</span></span> <span data-ttu-id="2eb09-122">Ezt ütemezheti úgy, hogy automatikusan történjen, vagy amikor a rendszer az eredeti adatok módosítását érzékeli.</span><span class="sxs-lookup"><span data-stu-id="2eb09-122">You can schedule this to happen automatically, or when the system detects a change to the original data.</span></span> <span data-ttu-id="2eb09-123">Egyes esetekben előfordulhat, hogy manuálisan újra létre kell hoznia a nézetet.</span><span class="sxs-lookup"><span data-stu-id="2eb09-123">In some cases it might be necessary to regenerate the view manually.</span></span> <span data-ttu-id="2eb09-124">Az ábra a Materialized View minta használatának példáját tartalmazza.</span><span class="sxs-lookup"><span data-stu-id="2eb09-124">The figure shows an example of how the Materialized View pattern might be used.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="2eb09-126">Problémák és megfontolandó szempontok</span><span class="sxs-lookup"><span data-stu-id="2eb09-126">Issues and considerations</span></span>
