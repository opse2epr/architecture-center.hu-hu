# <a name="bulkhead-pattern"></a><span data-ttu-id="e1083-104">Válaszfal minta</span><span class="sxs-lookup"><span data-stu-id="e1083-104">Bulkhead pattern</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="e1083-108">Kontextus és probléma</span><span class="sxs-lookup"><span data-stu-id="e1083-108">Context and problem</span></span>

<span data-ttu-id="e1083-109">Egy felhőalapú alkalmazás több szolgáltatást is tartalmazhat, és minden szolgáltatás egy vagy több fogyasztóval is rendelkezhet.</span><span class="sxs-lookup"><span data-stu-id="e1083-109">A cloud-based application may include multiple services, with each service having one or more consumers.</span></span> <span data-ttu-id="e1083-110">Ha egy szolgáltatás túl van terhelve vagy meghibásodik, az minden fogyasztót érint.</span><span class="sxs-lookup"><span data-stu-id="e1083-110">Excessive load or failure in a service will impact all consumers of the service.</span></span>

<span data-ttu-id="e1083-111">Továbbá egy fogyasztó egyszerre több szolgáltatáshoz is küldhet kéréseket, és minden kéréshez erőforrásokat használ fel.</span><span class="sxs-lookup"><span data-stu-id="e1083-111">Moreover, a consumer may send requests to multiple services simultaneously, using resources for each request.</span></span> <span data-ttu-id="e1083-112">Ha egy fogyasztó kérést küld egy rosszul konfigurált vagy nem válaszoló szolgáltatásnak, előfordulhat, hogy az ügyfél kérése által felhasznált erőforrások hogy nem szabadulnak fel megfelelő időn belül.</span><span class="sxs-lookup"><span data-stu-id="e1083-112">When the consumer sends a request to a service that is misconfigured or not responding, the resources used by the client's request may not be freed in a timely manner.</span></span> <span data-ttu-id="e1083-113">Ahogy újabb kérések érkeznek a szolgáltatáshoz, ezek az erőforrások kimerülhetnek.</span><span class="sxs-lookup"><span data-stu-id="e1083-113">As requests to the service continue, those resources may be exhausted.</span></span> <span data-ttu-id="e1083-114">Kimerülhet például az ügyfél kapcsolatkészlete.</span><span class="sxs-lookup"><span data-stu-id="e1083-114">For example, the client's connection pool may be exhausted.</span></span> <span data-ttu-id="e1083-115">Ezen a ponton a fogyasztó más szolgáltatásokhoz kérések érinti.</span><span class="sxs-lookup"><span data-stu-id="e1083-115">At that point, requests by the consumer to other services are affected.</span></span> <span data-ttu-id="e1083-116">Végül a fogyasztó már más szolgáltatásoknak sem fog tudni kéréseket küldeni, nem csak az eredeti nem válaszoló szolgáltatásnak.</span><span class="sxs-lookup"><span data-stu-id="e1083-116">Eventually the consumer can no longer send requests to other services, not just the original unresponsive service.</span></span>

<span data-ttu-id="e1083-117">Ugyanez az erőforrásfogyás a több fogyasztóval rendelkező szolgáltatásokat is érinti.</span><span class="sxs-lookup"><span data-stu-id="e1083-117">The same issue of resource exhaustion affects services with multiple consumers.</span></span> <span data-ttu-id="e1083-118">Az egy ügyféltől nagy számban érkező kérések kimeríthetik a szolgáltatás elérhető erőforrásait.</span><span class="sxs-lookup"><span data-stu-id="e1083-118">A large number of requests originating from one client may exhaust available resources in the service.</span></span> <span data-ttu-id="e1083-119">Így a többi fogyasztó tudja igénybe venni a szolgáltatást, ez pedig lépcsőzetesen terjedő hibákhoz vezet.</span><span class="sxs-lookup"><span data-stu-id="e1083-119">Other consumers are no longer able to consume the service, causing a cascading failure effect.</span></span>

## <a name="solution"></a><span data-ttu-id="e1083-120">Megoldás</span><span class="sxs-lookup"><span data-stu-id="e1083-120">Solution</span></span>

<span data-ttu-id="e1083-121">Particionálja a szolgáltatáspéldányokat különböző csoportokba a fogyasztók mennyisége és a rendelkezésre állási követelmények alapján.</span><span class="sxs-lookup"><span data-stu-id="e1083-121">Partition service instances into different groups, based on consumer load and availability requirements.</span></span> <span data-ttu-id="e1083-122">Ez a tervezés segít a hibák elkülönítésében, és meghibásodás esetén is biztosítja, hogy a szolgáltatás bizonyos fogyasztók számára elérhető legyen.</span><span class="sxs-lookup"><span data-stu-id="e1083-122">This design helps to isolate failures, and allows you to sustain service functionality for some consumers, even during a failure.</span></span>

<span data-ttu-id="e1083-123">A fogyasztók szintén particionálhatják az erőforrásokat, így az egyik szolgáltatás hívásához használt erőforrások nem befolyásolják a másik szolgáltatás hívásához használtakat.</span><span class="sxs-lookup"><span data-stu-id="e1083-123">A consumer can also partition resources, to ensure that resources used to call one service don't affect the resources used to call another service.</span></span> <span data-ttu-id="e1083-124">Például egy több szolgáltatást is hívó fogyasztóhoz szolgáltatásonként külön kapcsolatkészlet rendelhető hozzá.</span><span class="sxs-lookup"><span data-stu-id="e1083-124">For example, a consumer that calls multiple services may be assigned a connection pool for each service.</span></span> <span data-ttu-id="e1083-125">Ha egy szolgáltatás kezd meghibásodni, ez csak az adott szolgáltatáshoz hozzárendelt kapcsolatkészletet érinti, így a fogyasztó tovább használhatja a többi szolgáltatást.</span><span class="sxs-lookup"><span data-stu-id="e1083-125">If a service begins to fail, it only affects the connection pool assigned for that service, allowing the consumer to continue using the other services.</span></span>

<span data-ttu-id="e1083-126">A minta használata többek között a következő előnyökkel jár:</span><span class="sxs-lookup"><span data-stu-id="e1083-126">The benefits of this pattern include:</span></span>

- <span data-ttu-id="e1083-127">Elkülöníti a fogyasztókat és szolgáltatásokat a lépcsőzetesen terjedő hibáktól.</span><span class="sxs-lookup"><span data-stu-id="e1083-127">Isolates consumers and services from cascading failures.</span></span> <span data-ttu-id="e1083-128">A fogyasztókat vagy szolgáltatásokat érintő hibák elkülöníthetők a saját válaszfalukon belül, így megakadályozható a teljes megoldás meghibásodása.</span><span class="sxs-lookup"><span data-stu-id="e1083-128">An issue affecting a consumer or service can be isolated within its own bulkhead, preventing the entire solution from failing.</span></span>
- <span data-ttu-id="e1083-129">Lehetővé teszi a működésképesség bizonyos szintű fenntartását a szolgáltatás meghibásodása esetén is.</span><span class="sxs-lookup"><span data-stu-id="e1083-129">Allows you to preserve some functionality in the event of a service failure.</span></span> <span data-ttu-id="e1083-130">Az alkalmazás többi szolgáltatása és funkciója továbbra is működhet.</span><span class="sxs-lookup"><span data-stu-id="e1083-130">Other services and features of the application will continue to work.</span></span>
- <span data-ttu-id="e1083-131">Lehetővé teszi olyan szolgáltatások üzembe helyezését, amelyek eltérő minőségű szolgáltatást biztosítanak a felhasználó alkalmazások számára.</span><span class="sxs-lookup"><span data-stu-id="e1083-131">Allows you to deploy services that offer a different quality of service for consuming applications.</span></span> <span data-ttu-id="e1083-132">Beállítható egy magas prioritású fogyasztókészlet a magas prioritású szolgáltatások használatához.</span><span class="sxs-lookup"><span data-stu-id="e1083-132">A high-priority consumer pool can be configured to use high-priority services.</span></span>

<span data-ttu-id="e1083-133">Az alábbi ábra a különálló szolgáltatásokat hívó kapcsolatkészletek köré strukturált válaszfalakat ábrázol.</span><span class="sxs-lookup"><span data-stu-id="e1083-133">The following diagram shows bulkheads structured around connection pools that call individual services.</span></span> <span data-ttu-id="e1083-134">Ha az A szolgáltatás meghibásodik vagy egyéb hibát okoz, a kapcsolatkészlet elkülönítése miatt a hiba csak azokat a számítási feladatokat érinti, amelyek az A szolgáltatáshoz hozzárendelt szálkészletet használják.</span><span class="sxs-lookup"><span data-stu-id="e1083-134">If Service A fails or causes some other issue, the connection pool is isolated, so only workloads using the thread pool assigned to Service A are affected.</span></span> <span data-ttu-id="e1083-135">A B és C szolgáltatásokat használó számítási feladatokat ez nem befolyásolja, és megszakítás nélkül működhetnek tovább.</span><span class="sxs-lookup"><span data-stu-id="e1083-135">Workloads that use Service B and C are not affected and can continue working without interruption.</span></span>

## <a name="example"></a><span data-ttu-id="e1083-163">Példa</span><span class="sxs-lookup"><span data-stu-id="e1083-163">Example</span></span>

<span data-ttu-id="e1083-164">Az alábbi Kubernetes konfigurációs fájl egy egyetlen szolgáltatást futtató elkülönített tárolót hoz létre, amely saját processzor- és memória-erőforrásokkal, valamint korlátokkal rendelkezik.</span><span class="sxs-lookup"><span data-stu-id="e1083-164">The following Kubernetes configuration file creates an isolated container to run a single service, with its own CPU and memory resources and limits.</span></span>

  name: drone-management
spec:
  containers:
  - name: drone-management-container
    image: drone-service
    resources:
      requests:
        memory: "64Mi"
        cpu: "250m"
      limits:
        memory: "128Mi"
        cpu: "1"
